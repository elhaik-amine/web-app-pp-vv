Project Summary

The backend is a REST API for a home-services marketplace. Clients register, browse providers, create bookings, chat around bookings, report users, manage notifications, and buy platform tokens. Providers manage bookings and service flow. Admins moderate users, reports, bookings, and token activity.

Authentication

The API uses JWT bearer tokens. Client and provider login endpoints return an access token signed with JWT_SECRET and expiring in 7 days. The protect middleware verifies the token, loads the user from the database, and attaches req.user.

Access Token Rules

  • Header format: Authorization: Bearer <token>
  • 401 if header is missing, malformed, token invalid, or user no longer exists
  • 403 if role middleware rejects the current user
  • 403 if suspended user hits a route that also uses restricted

Password Reset Flow

  • POST /api/auth/forgot-password stores a 1-hour JWT in the user record
  • Email links to {FRONTEND_URL}/reset-password?token=...
  • POST /api/auth/reset-password validates both JWT signature and stored token match
  • Reset token is cleared after successful password reset

  "Authorization": "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

Common Response Patterns

Most controller responses use a JSON envelope with success, data, and sometimes message. Error handling is not fully uniform across explicit controller responses and fallback middleware.

200 / 201 Success
Usually { success: true, data, message? }
400 Bad Request
Manual validation failures, invalid state transitions, bad input
401 Unauthorized
Missing/invalid token or deleted user
403 Forbidden
Role restriction, suspended account, or ownership/access failure
404 Not Found
Missing resource; unmatched routes use fallback middleware shape
409 Conflict
Duplicate booking slot or duplicate review, inferred from DB unique constraints
422 Validation Error
Not used by mounted code
500 Internal Server Error
Usually { success: false, message: error.message }

Operational Notes

Socket.IO Events

  • join joins personal room user_{userId}
  • join:booking joins booking room booking_{bookingId}
  • Server emits events such as notification:new, booking:new, booking:confirmed, booking:started, booking:price_set, message:new, and token:updated

Filtering, Pagination, Sorting

  • No pagination was found on mounted endpoints
  • Provider listing filters by city and category_id
  • User/admin/report/booking listing endpoints support limited query filtering
  • Sorting is fixed in SQL and not configurable by query params
